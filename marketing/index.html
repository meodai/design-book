A reactive design-token framework

Tokens with relationships.

Most design tokens hold a value. The interesting ones hold a rule. Design Book lets your palette answer questions — which color reads best on this surface, which is closest to this brand, which is the perceptual average of those five — and recompute the moment the inputs change.

$ npm i design-book

TypeScript · ESM · one dependency (culori).

The premise

CSS can already modify a color — color-mix, oklch(from …), calc(), custom properties. That part is solved. What CSS cannot do — and will not, for a long time — is select a value from a set you defined.

Design Book treats your palette as a structured collection that can be queried. Six functions iterate the set and return a single member: the most readable, the most distant, the most chromatic, the closest match. When you add a color, the answers update. Swap a brand — every dependent token reflows.

bestContrastWith

bestContrastWith( target, scope, { not? } )

Find the most readable color from a palette for any given background. The function walks every member of the scope, measures WCAG contrast, returns the winner. Bind it to a surface token; the result tracks whatever the surface becomes.

Use for accessible UI text, dynamic badges, themable buttons, chart labels. 

// label auto-selects its readable color
ui.set('label', bestContrastWith(
  ref('ui.surface'),
  scope('brand'),
))

closestColor

closestColor( target, scope, { not? } )

Match an external color — a brand request, a Figma frame, a freshly sampled hue — to its nearest neighbour in your token set. Distance is measured in OKLab, so matches are perceptual, not RGB-shaped.

Use for normalising third-party colors into your palette, generating tinted assets, snapping eyedropper input to a system color. 

// snap an incoming color to a palette member
brand.set('logo', closestColor(
  color('#7f4dc4'),
  scope('palette'),
))

furthestFrom

furthestFrom( scope, { not? } )

The companion of closest. Given a scope, find the member that sits perceptually farthest from a chosen anchor. Maximum visual distance, by construction.

Use for accent picking, “opposite” tokens, chart series that mustn't read as siblings. 

// pick an accent guaranteed to read as “other”
ui.set('accent', furthestFrom(
  scope('brand'),
))

minContrastWith

minContrastWith( target, scope, { threshold?, not? } )

The opposite optimisation. Find the palette member with the lowest contrast against a background that still clears a configurable threshold. Subtle, but legible.

Use for captions, helper text, disabled states, borders that exist but don't shout. 

// the quietest member that still clears AA
ui.set('caption', minContrastWith(
  ref('ui.surface'),
  scope('neutral'),
  { threshold: 4.5 },
))

averageColor

averageColor( scope, { space? = 'oklab' } )

Reduce a palette to its perceptual mean. Averaging happens in OKLab by default — the result is the color that feels like the centre of mass, not the math of RGB sludge.

Use for background tints, neutral derivations, brand-aware greys, fallback colors. 

// the perceptual centre of a palette
neutral.set('bg', averageColor(
  scope('brand'),
  { space: 'oklab' },
))

mostVivid

mostVivid( scope, { not? } )

Pick the highest-chroma member of a scope. Vividness is measured in OKLCh — independent of lightness — so a fluorescent yellow and a deep violet compete on the same axis.

Use for hero accents, focus indicators, spot colors, status signals. 

// the highest-chroma member of a set
ui.set('accent', mostVivid(
  scope('brand'),
))

Also included

Eight modifier functions. CSS can already do most of these natively — we ship them so a single token graph can describe a whole system without falling back to strings.

  • colorMix Blend two colors in any space.
  • lighten Increase OKLCh lightness.
  • darken Decrease OKLCh lightness.
  • shade Mix toward black, perceptually.
  • relativeTo Modify a color in a chosen channel.
  • spacingScale Modular spacing steps.
  • typographyScale Type ramps from a ratio.
  • timing Animation curves & multiples.

Define the rules. Let the system do the picking.
