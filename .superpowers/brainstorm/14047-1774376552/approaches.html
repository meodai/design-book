Spreadsheet Editor: Which approach?

Three viable options for the token grid editor

A

Custom HTML Table + CodeMirror (DIY)

0 KB added — reuse everything you already have

  • HTML <table> with Name / Formula / Value columns
  • Click a formula cell → CodeMirror activates in-place with full autocomplete
  • Tab bar for scope switching (trivial HTML/CSS)
  • Color swatches, error highlighting, inherit — all reused from current editor
  • ~150 LOC for keyboard nav (arrows, Enter, Esc, Tab)

Pros

  • Zero new deps
  • Full control over UX
  • Reuse existing CodeMirror setup
  • Tailored to token editing

Cons

  • Must build keyboard nav
  • No multi-cell select/drag-fill
  • Focus management DIY
B

Jspreadsheet CE

~35 KB gzip — lightweight spreadsheet with built-in tabs

  • MIT, vanilla JS, no framework dependency
  • Built-in worksheet tabs (unique among lightweight options)
  • Custom cell editor API — can embed CodeMirror
  • Built-in autocomplete column type (but likely want CodeMirror's)
  • Excel-like keyboard nav out of the box

Pros

  • Tabs built-in
  • Keyboard nav free
  • Custom editors supported
  • Spreadsheet feel out of box

Cons

  • A1 cell reference model (awkward for key-value)
  • New dependency
  • Fighting lib abstractions
  • CE docs mix with Pro docs
C

AG Grid Community

~200 KB gzip — battle-tested grid, best custom editor API

  • MIT, framework-agnostic, huge ecosystem
  • Most mature custom cell editor interface
  • Excellent keyboard navigation
  • cellEditorPopup for overlay editing
  • No built-in tabs (build tab bar yourself)

Pros

  • Most polished editing UX
  • Huge community
  • Handles edge cases
  • Virtual scrolling

Cons

  • ~200KB for a 2-column grid
  • No tabs
  • Overkill for this use case
  • Enterprise features tempting but paywalled
